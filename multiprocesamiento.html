💻 Sistemas Operativos

👥 Sistema Multiusuario

Concepto Fundamental: Un sistema operativo multiusuario permite que múltiples usuarios accedan y utilicen simultáneamente los recursos de una sola computadora o servidor, cada uno con su propia sesión y datos protegidos.

Funcionamiento Técnico:

  • El sistema operativo asigna slots de tiempo (time slicing) a cada usuario para que ejecute sus procesos.
  • Cada usuario tiene un identificador único (UID) y una cuenta de seguridad independiente.
  • La memoria se particiona para aislar el espacio de cada usuario evitando que interfieran entre sí.
  • El kernel gestiona permisos de acceso y protege los datos privados de cada usuario.

Características Principales:

  • Seguridad: Cada usuario tiene credenciales de autenticación (usuario/contraseña).
  • Privacidad: Los archivos de un usuario no son visibles por defecto para otros usuarios.
  • Concurrencia Controlada: Múltiples procesos se ejecutan en paralelo de forma ordenada.
  • Administración de Recursos: Cuotas de disco, CPU y memoria por usuario.
  • Sesiones Independientes: Cada usuario tiene su propio desktop virtual, variables de entorno y preferencias.

Tipos de Acceso Multiusuario:

  • Local: Múltiples usuarios en terminales conectadas directamente al servidor.
  • Remoto: Usuarios accediendo vía SSH, RDP o protocolos de red desde máquinas distintas.
  • Híbrido: Combinación de acceso local y remoto en entornos corporativos.

Ejemplos de Sistemas Multiusuario: Linux, Unix, macOS Server, Windows Server, Solaris. En la actualidad, incluso Windows 10/11 soporta multiusuario aunque típicamente se usa en modo monousuario.

⚙️ Multiprocesamiento

Concepto Fundamental: El multiprocesamiento se refiere al uso de dos o más procesadores (CPUs) dentro de una misma computadora para ejecutar procesos e hilos de manera verdaderamente paralela, no solo intercalada.

Diferencia con Multitarea:

  • Multitarea: Un solo procesador cambia rápidamente entre tareas (context switching). Es simultánea en apariencia pero secuencial en realidad.
  • Multiprocesamiento: Múltiples procesadores ejecutan verdaderamente en paralelo. Dos tareas se ejecutan simultáneamente en dos CPUs diferentes.

Funcionamiento Técnico:

  • Cada CPU tiene su propia cache y registros, conectados a una memoria compartida.
  • El sistema operativo distribuye procesos entre CPUs según disponibilidad y políticas de scheduling.
  • Se requiere sincronización mediante candados (locks) y semáforos para acceso a recursos compartidos.
  • El kernel mantiene una cola de procesos listos para ejecutar en cada CPU.

Tipos de Multiprocesamiento:

  • Simétrico (SMP): Todos los procesadores son iguales y pueden ejecutar cualquier proceso. El más común en sistemas modernos.
  • Asimétrico: Un procesador (master) controla a otros (slave) que solo ejecutan tareas específicas asignadas.

Ventajas del Multiprocesamiento:

  • Rendimiento superior: Ejecución verdaderamente paralela aumenta throughput.
  • Escalabilidad: Se pueden agregar más CPUs para mejorar rendimiento.
  • Resiliencia: Si falla una CPU, otras continúan funcionando.
  • Ideal para aplicaciones intensivas (bases de datos, servidores web, análisis científico).

Aplicaciones Modernas: Servidores de datos centers, computadoras científicas, sistemas HPC (High Performance Computing), videojuegos, procesamiento de imágenes, inteligencia artificial.

⚠️ Compatibilidad en Sistemas Operativos

Concepto Fundamental: La compatibilidad se refiere a la capacidad de un sistema operativo de ejecutar software diseñado para versiones anteriores o de diferentes sistemas, así como soportar hardware variado.

Tipos de Compatibilidad:

  • Compatibilidad Hacia Atrás (Backward Compatibility): Una versión nueva del SO puede ejecutar programas diseñados para versiones antiguas.
  • Compatibilidad Hacia Adelante (Forward Compatibility): Software nuevo puede funcionar en sistemas antiguos (raro de lograr).
  • Compatibilidad de Hardware: Soporte de drivers para dispositivos periféricos variados (impresoras, tarjetas de red, etc.).
  • Compatibilidad de Binarios: La capacidad de ejecutar código compilado sin recompilación.

Problemas de Compatibilidad Comunes:

  • Cambios de Arquitectura: Transición de 32 a 64 bits. Software de 32 bits puede funcionar en sistemas 64 bits pero con limitaciones.
  • Cambios de API: Las funciones disponibles para programadores cambian entre versiones, rompiendo código antiguo.
  • Cambios de Sistema de Archivos: De FAT32 a NTFS. Sistemas antiguos no pueden leer archivos en formatos nuevos.
  • Obsolescencia de Periféricos: Drivers antiguos dejan de funcionar en hardware moderno.
  • Dependencias de Librerías: Software que requiere versiones específicas de DLLs que no están disponibles.

Ejemplos de Falta de Compatibilidad:

  • Software de Windows XP no funciona en Windows 11 64 bits sin modo de compatibilidad.
  • Impresoras antiguas no tienen drivers para sistemas operativos recientes.
  • Videojuegos de arquitectura 16 bits no se ejecutan en sistemas modernos.
  • Aplicaciones macOS de Intel no funcionan nativamente en Apple Silicon sin traducción.

Impacto Empresarial: Migración costosa de sistemas antiguos, mantenimiento de múltiples versiones, pérdida de productividad durante transiciones.

Soluciones de Compatibilidad:

  • Modo de Compatibilidad: Windows permite ejecutar programas antiguos con configuraciones especiales.
  • Máquinas Virtuales: Emulación de SO antiguos para ejecutar software legacy.
  • Wine/Proton: Capas de compatibilidad para ejecutar Windows en Linux.
  • Traducción Dinámica: Apple Rosetta traduce código Intel a ARM en tiempo real.

📊 Tipos de Sistemas Operativos

Clasificación por Número de Usuarios:

  • Monousuario: Solo un usuario puede usar el sistema a la vez. Ejemplo: Windows 10, macOS.
  • Multiusuario: Múltiples usuarios acceden simultáneamente. Ejemplo: Linux Server, Unix.

Clasificación por Número de Tareas:

  • Monotarea: Un programa se ejecuta a la vez. Ejemplo: DOS.
  • Multitarea: Múltiples programas se ejecutan simultáneamente (intercalados o en paralelo). Ejemplo: Todos los SO modernos.

Clasificación por Procesamiento:

  • Uniprocesador: Una sola CPU. Común en computadoras personales antiguas.
  • Multiprocesador: Múltiples CPUs. Servidores, computadoras gaming modernas.

Clasificación por Tiempo de Respuesta:

  • Batch: Procesa trabajos en lotes sin interacción del usuario. Ej: sistemas de nómina antiguos.
  • Interactivo: Responde inmediatamente a entrada del usuario. Ej: Windows, Linux Desktop.
  • Tiempo Real: Garantiza respuesta dentro de plazos estrictos. Ej: Sistemas de control industrial, vehículos autónomos.

Gestión de Recursos del Sistema Operativo:

  • Gestión de Procesos: Crear, ejecutar, pausar y terminar procesos. Asignación de CPU según prioridad.
  • Gestión de Memoria: Asignación de RAM a procesos, proteción de memoria, manejo de memoria virtual.
  • Sistema de Archivos: Organización jerárquica de carpetas y archivos en disco duro.
  • Gestión de E/S: Control de periféricos (impresoras, teclado, ratón, red, USB).
  • Seguridad: Autenticación de usuarios, control de acceso, encriptación de datos.
  • Red y Comunicaciones: Protocolo TCP/IP, manejo de conexiones, firewall.

SO Más Utilizados Actualmente:

  • Windows: Desktops y servidores empresariales. Cuota de mercado: ~75% en desktops.
  • macOS: Computadoras Apple. Cuota: ~15% en desktops, muy popular en creativos.
  • Linux: Servidores, sistemas embebidos, data centers. Cuota en servidores: ~70%.
  • iOS/Android: Móviles y tablets. Combinados: ~99% del mercado mobile.

📱 Usos y Ámbitos de los Sistemas Operativos

Computadoras Personales (Desktop/Laptop):

  • Windows: Trabajo de oficina, gaming, desarrollo.
  • macOS: Diseño gráfico, edición de video, desarrollo.
  • Linux: Desarrollo de software, servidores locales, usuarios técnicos.

Dispositivos Móviles:

  • iOS (Apple): Iphones, Ipads. Enfoque en seguridad y experiencia.
  • Android (Google): Mayor cuota de mercado, diversos fabricantes.
  • Características: Optimización para batería, touchscreen, ubicación.

Servidores Empresariales:

  • Linux Server: Apache, Nginx, bases de datos MySQL/PostgreSQL.
  • Windows Server: Integración con Active Directory, Microsoft Exchange, SQL Server.
  • Requieren uptime 24/7, redundancia, alta seguridad.

Internet de las Cosas (IoT):

  • Sistemas embebidos: Linux embebido, FreeRTOS.
  • Aplicaciones: Termostatos inteligentes, sensores, smartwatches.
  • Requisitos: Bajo consumo, tamaño reducido, funcionamiento autónomo.

Sistemas de Tiempo Real:

  • Control industrial: Robots, máquinas de manufactura.
  • Automotriz: Sistemas ABS, control de motor.
  • Aviación: Sistemas de navegación, piloto automático.
  • Médico: Monitores cardíacos, ventiladores mecánicos.

Centros de Datos:

  • Virtualización: VMware, Hyper-V, KVM.
  • Contenedorización: Docker, Kubernetes.
  • Computación en la nube: AWS, Azure, Google Cloud.

Otras Aplicaciones: Televisores inteligentes, consolas de videojuegos, sistemas de entretenimiento automotriz, cámaras de seguridad, enrutadores de red.

🌐 Aplicaciones Actuales de los S.O.

Gaming y Entretenimiento:

  • Plataformas: PC (Windows/Linux), PlayStation, Xbox, Nintendo, móviles.
  • Requisitos: Optimización de GPU, baja latencia, almacenamiento de alta velocidad (SSD).
  • Ejemplos: Fortnite, Call of Duty, Elden Ring en múltiples plataformas.

Computación en la Nube:

  • Infraestructura: AWS (Elastic Compute Cloud), Microsoft Azure, Google Cloud Platform.
  • Modelos: IaaS (máquinas virtuales), PaaS (plataformas), SaaS (software).
  • Beneficios: Escalabilidad, pago por uso, mantenimiento reducido.
  • Datos: 60% de aplicaciones empresariales corren en la nube actualmente.

Inteligencia Artificial y Machine Learning:

  • Frameworks: TensorFlow, PyTorch, scikit-learn.
  • Sistemas: Reconocimiento facial, procesamiento de lenguaje natural (ChatGPT, Copilot).
  • Requisitos: GPUs potentes, CPUs multicore, grandes cantidades de RAM.

Ciencia e Investigación:

  • Computación de Alto Rendimiento (HPC): Simulaciones climáticas, investigación nuclear, cálculos astrofísicos.
  • Bioinformática: Análisis de genomas, modelado de proteínas.
  • Plataformas: Supercomputadoras con Linux, clusters distribuidos.

Industria y Manufactura:

  • Automatización: Robots industriales, sistemas de control, IoT.
  • Industria 4.0: Máquinas conectadas, análisis de datos en tiempo real.
  • Sistemas: Tiempo real con determinismo garantizado.

Finanzas y Criptografía:

  • Trading algorítmico: Análisis de mercados en milisegundos.
  • Blockchain: Bitcoin, Ethereum, criptomonedas.
  • Requisitos: Seguridad extrema, rendimiento máximo, disponibilidad 24/7.

Streaming y Redes Sociales:

  • Plataformas: Netflix (Linux), YouTube (Google), TikTok.
  • Infraestructura: Millones de servidores distribuidos globalmente.

🪟 Evolución Histórica de Windows

Era 1.0 - 3.1 (1985-1994):

  • Windows 1.0: Interfaz gráfica sobre MS-DOS. No era un SO completo, sino un entorno gráfico.
  • Windows 3.0/3.1: Primera versión exitosa con 10 millones de copias vendidas. Introduced TrueType fonts y multimedia.
  • Arquitectura: 16-bit, dependiente de DOS, 2-4 MB de RAM.

Era 95/98 (1995-2000):

  • Windows 95: Revolucionario con menú Inicio, barra de tareas, Plug & Play. Vendió 40 millones de copias en 4 años.
  • Windows 98: Incorporó USB, FAT32, mayores velocidades. Más estable pero aún híbrido 16/32 bits.
  • Windows Me (2000): Versión intermedia, muchos problemas técnicos, es considerada un fracaso.

Era NT (2000-2007):

  • Windows 2000: Primera versión completamente NT para consumidor. Muy estable, requería más recursos.
  • Windows XP (2001): Unificó línea profesional y doméstica. Fue la versión más exitosa de Windows con 400+ millones de copias. Soporte hasta 2014.
  • Características: Cifrado de archivos, hibernación, restauración del sistema, mejor multimedia.

Era Post-XP (2007-2015):

  • Windows Vista (2007): Enfoque en seguridad (UAC - User Account Control). Pero requería más recursos, problemas de drivers, poco popular.
  • Windows 7 (2009): Rediseño de Vista, mucho más rápido. Segunda versión más exitosa después de XP. Soporte extendido hasta 2020.
  • Windows 8 (2012): Interfaz "Modern" táctil, problemas de UX para usuarios de escritorio, fue impopular.
  • Windows 8.1 (2013): Intento de corrección, permitía usar modo desktop.

Era Moderna (2015-Presente):

  • Windows 10 (2015): Retorno a escritorio clásico. Introducción de "Windows as a Service" (actualizaciones continuas).
  • Características: Cortana (asistente virtual), Edge (nuevo navegador), soporte 32 y 64 bits.
  • Versión universal para PC, tablets, y dispositivos IoT bajo el proyecto "OneCore".
  • Windows 11 (2021): Interfaz modernizada con esquinas redondeadas, solo 64 bits, requiere TPM 2.0 y Secure Boot.
  • Características: Integración profunda con Android, Microsoft Teams, mejoras de rendimiento.

Tendencias Futuras:

  • Enfoque en seguridad con requisitos más estrictos de hardware.
  • Convergencia de dispositivos (PC, tablet, teléfono en uno).
  • Computación en la nube más integrada (Microsoft 365).
  • Sistemas operativos cada vez más ligeros en dispositivos móviles.
